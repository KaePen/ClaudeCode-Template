# Python開発ルール

## 出力原則

- コード直接出力、説明最小化
- 手順説明・事前解説禁止
- エラー修正は即座にコードで対応
- タスク分解時にドキュメント作成タスクを含めない
- レポートの結果報告方法

## タスク管理ルール

### 禁止タスク

以下のタスクは作成しない:

- ドキュメント作成
- README更新
- コメント追加
- docstring追加（コード作成時に同時に含める）
- 使用方法説明文書の指示なし作成
- 変更履歴文書の指示なし作成

### 許可タスク

- コード実装（docstring含む）
- テスト作成
- バグ修正
- リファクタリング
- 設定ファイル作成

### タスク分解の原則

1. コードとdocstringは常に1つのタスクとして扱う
2. ドキュメントが必要な場合は、コード実装タスク内で完結させる
3. 「ドキュメント」「README」「説明」などの単独タスクは作成しない
4. コードを修正する場合、必要時以外は新規作成ではなく既存修正を基本とする
5. 検証用などでコードを一時的に作成する場合は、使用後はアーカイブフォルダを作成しそこに移動し保管すること

### 試験結果報告方法の原則

- repotsフォルダに出力する
- レポートの結果にはプログラムの説明は不要
- 操作方法と結果のみを記載

## コーディング規約

### 必須事項

- PEP8厳守（79文字制限）
- `from __future__ import annotations` 使用
- 型ヒント必須（小文字型: `list`, `dict`, `any`）
- Googleスタイルdocstring必須（コード作成と同時）
- 全コメント日本語

### import規則

from __future__ import annotations

# 標準ライブラリ

import os

# サードパーティ

import numpy as np

# ローカル

from .module import function

### 禁止事項

- 遅延import（関数内import）
- bare except
- マジックナンバー
- 大文字型ヒント（List, Dict等）
- コードとドキュメントの分離タスク化

## 応答パターン

- 通常: コード即出力（docstring込み）
- 不明点: 1質問→即コード
- 修正: 修正コード+変更点1行
